# Plan: Standalone Badminton Tracker (No Google Sheets)

## Goal

Create a new standalone version (`app_standalone.py`) that uses only the local SQLite DB — no Google Sheets dependency at all. The existing `app.py` remains unchanged for users who want Sheets sync.

## Key Features

1. **Fixed 12-player roster** (โรจน์, น้อย, ภูมี, ป๊อป ภู, คะน้า, จืด, เกียรติ, อองรี, ป๊อป, น้อต, ต้น, ทรัมป) with ability to add more players on the day
2. **Editable games** — tap to change which players played each game
3. **Payment checkbox** in Split view to mark players as Paid
4. **Per-day summary** in Split view
5. **CSV export** of all records

## Approach

Instead of modifying the existing `app.py` (which is tied to Google Sheets), create `app_standalone.py` as a clean rewrite. Both apps share the same `summary_db.py`.

### Database Schema

Reuse the existing `summary_db.py` schema — `sessions`, `session_players`, `session_games` — which already supports everything needed.

### UI Tabs

| Tab | Content |
|-----|---------|
| 🏟️ Live Tracker | Attendance toggles for fixed roster + "Add player" input, game recording with edit/delete |
| 🧾 Split | Per-player cost breakdown with Paid checkbox, daily summary, "Submit day" |
| 📊 History | Same as current — expandable per-session cards, player totals, chart |
| 📤 Export | CSV download of all sessions, or per-player |

### Editable Games

Instead of the current "add game → add to sheet → rerun" flow, games live in `st.session_state` and are only persisted to SQLite on "Submit day". Each game in the list has an "Edit" button that opens inline fields to change players and shuttles.

### Payment Flow

The Split view shows each player's cost with a checkbox. When checked, `payment_status` flips to "Paid" in the local DB immediately.

### CSV Export

A new view with:
- Download all sessions as CSV
- Download per-player summary as CSV
- Uses `st.download_button` with pandas `to_csv()`

### Files

| File | Change |
|------|--------|
| `app_standalone.py` | **New** — standalone app |
| `summary_db.py` | Add `update_payment_status()` function |
